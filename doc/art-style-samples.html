Bytewars — art style comparison

Same two chassis (vacuum-class, lawnbot-class) rendered three ways. All three are pure DOM+CSS; no images, no sprites. Open this file in a browser to view.

A — Flat vector (currently locked)

Thick outline, 5–7 flat colors, no gradients. Silhouette-first. What setting.md §4 specifies today.

A / flat vector

B — Anime-flavored (same DOM+CSS budget)

Silhouette-first, but with cel-shading (2 tones + highlight), richer palette, warmer stage lighting. Still no sprites. Reuses the current rendering pipeline.

B / anime flavor

C — Full anime (approximation)

CSS pushed as far as it goes: gradient shading, rim light, glowing eyes, atmospheric stage. Closest DOM+CSS can get to cel-shaded anime — real Dragon's Blood / Avatar fidelity needs hand-drawn or rendered sprites, not divs.

C / anime approx

How to read this. A is what setting.md commits to today. B is the cheap upgrade — same tech stack, same silhouette rules, richer palette and cel-shading. C is the ceiling of what DOM+CSS can do before you'd need a real art pipeline; it's not what true anime art looks like, it's what a stylized mockup of it looks like built from divs. If C is close to what you want, we should talk about whether it's worth reopening the rendering decision in architecture.md.
